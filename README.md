# CodingWeb MVP 使用说明

这是一个最小可用版的自定义刷题网站，当前只支持 Java。

你可以先把它理解成一个本地运行的小型力扣：

- 可以创建自己的题目。
- 可以配置样例测试用例和隐藏测试用例。
- 可以在网页里写 Java 代码。
- 可以运行样例。
- 可以正式提交，让系统用全部测试用例判题。

## 1. 启动前要确认什么

这个项目现在不需要安装第三方依赖，但需要电脑里已经有两个东西：

- Node.js：用来启动网站后端。
- Java JDK：用来编译和运行你提交的 Java 代码。

在 PowerShell 里进入项目目录：

```powershell
Set-Location -LiteralPath "F:\项目\CodingWeb"
```

然后分别输入：

```powershell
node --version
javac -version
java -version
```

如果能看到版本号，就说明环境基本可用。例如：

```text
v20.16.0
javac 22.0.2
java version "22.0.2"
```

如果提示“不是内部或外部命令”，说明对应工具还没有安装，或者没有配置到系统环境变量。

## 2. 如何启动网站

还是在项目目录 `F:\项目\CodingWeb` 下运行：

```powershell
npm start
```

正常情况下会看到类似输出：

```text
CodingWeb MVP started at http://127.0.0.1:3000
```

这句话的意思是：网站已经启动，浏览器访问下面这个地址即可：

```text
http://127.0.0.1:3000
```

注意：运行 `npm start` 的 PowerShell 窗口不要关。关掉窗口后，网站服务也会停止。

## 3. 如果 3000 端口被占用

如果你的电脑上已经有别的软件占用了 3000 端口，项目会自动尝试使用 3001 到 3010。

实际启动地址会写在这个文件里：

```text
tmp/server.json
```

可以用下面命令查看：

```powershell
Get-Content -LiteralPath "tmp\server.json"
```

你会看到类似内容：

```json
{
  "url": "http://127.0.0.1:3000",
  "port": 3000,
  "pid": 29004
}
```

其中 `url` 就是当前网站地址。

## 4. 如何停止网站

如果你是用 `npm start` 启动的，并且 PowerShell 窗口还开着：

1. 点击那个 PowerShell 窗口。
2. 按 `Ctrl + C`。
3. 如果提示是否终止，输入 `Y` 后回车。

如果服务是在后台启动的，可以先查看 `tmp/server.json` 里的 `pid`，再执行：

```powershell
Stop-Process -Id 29004
```

上面的 `29004` 只是例子，要替换成你自己文件里的 `pid`。

## 5. 页面怎么用

打开网站后，页面大致分三块：

- 左侧：题目列表。
- 中间：题目详情、题目解析、提交记录、代码编辑器、运行结果。
- 右侧：题目管理表单，可以通过顶部“题目管理”按钮展开或收起。

### 做一道题

1. 在左侧点击一道题，例如“两数之和”。
2. 中间会显示题目描述、输入说明、输出说明和样例。
3. 在 Java 代码框里写代码。
4. 点击“运行样例”，系统只会跑样例测试用例。
5. 如果样例没问题，再点击“提交”，系统会跑全部测试用例，包括隐藏用例。
6. 题目描述区域顶部的“提交记录”会保存当前题目的运行和提交结果。
7. 如果一次运行有多个用例，结果区一次只显示一个用例，可以点击“上一个”“下一个”切换查看。

“运行样例”和“提交”的区别：

- 运行样例：用于快速检查，能看到样例输入、期望输出、实际输出。
- 正式提交：用于最终判题，隐藏用例不会暴露输入输出，只告诉你通过数量。

### 查看提交记录

1. 在题目描述区域顶部点击“提交记录”标签。
2. 每条记录会显示题目名、判题状态、通过数量、耗时和提交时间。
3. 点击某条记录，可以查看这次提交的用例结果和代码。
4. 如果刚提交后没看到记录，可以点击“刷新记录”。

### 查看题目解析

1. 在题目描述区域顶部点击“题目解析”标签。
2. 当前版本先显示解析占位内容。
3. 后续可以把手写解析或大模型生成的解析保存到题目里。

### 新建自己的题目

1. 点击右上角“新建题目”。
2. 在右侧填写题目标题、难度、标签。
3. 填写题目描述、输入说明、输出说明、数据范围。
4. 填写 Java 模板。模板里必须有 `public class Main`。
5. 添加测试用例。
6. 勾选“样例用例”的用例会显示给做题用户。
7. 不勾选“样例用例”的用例就是隐藏用例。
8. 点击“保存题目”。
9. 保存后，左侧题目列表会出现新题目，右侧题目管理会自动收起。

如果只是想修改当前题目，可以点击顶部“题目管理”按钮重新展开右侧表单。

建议每道题至少配置：

- 1 个样例用例。
- 1 个隐藏用例。

这样才能同时测试“用户能看到的例子”和“正式提交时的隐藏检查”。

### 恢复示例题

如果你不小心把“两数之和”示例题删掉了，可以点击页面右上角“恢复示例题”。

这个按钮会做两件事：

- 如果示例题不存在，就重新创建“两数之和”。
- 如果示例题已经存在，就不会重复创建。

## 6. Java 代码有什么要求

当前 MVP 只支持一种写法：

```java
import java.util.*;

public class Main {
    public static void main(String[] args) {
        Scanner scanner = new Scanner(System.in);
        // 从标准输入读取数据
        // 向标准输出打印答案
    }
}
```

重点要求：

- 类名必须是 `Main`。
- 必须写成 `public class Main`。
- 从标准输入读取，也就是用 `Scanner`、`BufferedReader` 等读取。
- 向标准输出打印，也就是用 `System.out.println` 等输出。
- 不要写 `package xxx;`。
- 不要读写本机文件。
- 不要创建系统进程或访问网络。

## 7. 判题结果怎么看

常见结果含义：

- `通过`：答案正确。
- `答案错误`：程序正常运行了，但输出和期望输出不一致。
- `编译错误`：Java 代码没有通过 `javac` 编译。
- `运行错误`：程序运行时崩溃，例如数组越界。
- `运行超时`：程序超过了题目的时间限制。

如果出现错误，先看结果区域里的错误信息。编译错误通常会指出具体哪一行有问题。

## 8. 数据保存在哪里

当前版本使用本地 JSON 文件保存数据：

```text
data/db.json
```

这里保存了：

- 题目。
- 测试用例。
- 提交记录。

一般不要手动改这个文件。后续如果需要重置数据，可以让我来操作，避免格式写错导致网站无法读取。

## 9. 当前功能

- 题目列表。
- 题目详情。
- 新建、编辑、删除自定义题目。
- 恢复内置示例题。
- 配置样例用例和隐藏用例。
- Java 代码编辑。
- 运行样例。
- 正式提交并使用全部测试用例判题。
- 查看提交记录和提交详情。
- 本地 JSON 存储题目和提交记录。

## 10. 当前限制

这是本地开发用的 MVP，不是正式线上版本。

当前 Java 代码是在本机进程中编译运行的，只做了基础限制：

- 编译超时限制。
- 运行超时限制。
- 输出大小限制。
- 部分危险 Java API 拦截。

后续要做成更安全的正式版本，需要按 `刷题网站流程文档.md` 中的流程，把判题迁移到 Docker 隔离环境。
